The Problem

Fit is the #1 reason people return clothes online.

The Status Quo

Statistical guessing

Every VTO product correlates historical data or warps photos. None simulate actual cloth mechanics.

The Result

30–40% return rates

Users receive garments that don't fit. Brands absorb return costs. Trust in online fashion breaks down.

The Gap

No physics ground truth

Statistical models need accurate fit labels to train on. Those labels don't exist anywhere today.

The Output

Not "you might like size M."

A deterministic, explainable, physically-grounded fit verdict.

{ "fit": true, "confidence": 1.0, "strain_map": [ { "region": "chest_front", "delta_mm": +12.4, "severity": "green" }, { "region": "shoulder_left", "delta_mm": +24.1, "severity": "green" }, { "region": "waist", "delta_mm": -8.2, "severity": "green" } ] }

Negative = body larger than garment (too tight). Positive = garment larger (ease). Every verdict comes from real simulation — nothing hardcoded.

The Full Vision

How the product works end-to-end

Phase 1 — MVP

Measurements → Fit Verdict

User enters 6 measurements. Forma generates their body mesh, finds nearest reference bodies, interpolates fit from precomputed simulations. Returns: fits / doesn't fit + key measurements.

Phase 2 — Full Product

Desired Fit → Garment Discovery

User clicks/drags on their mannequin to define how they want a garment to fit. Pattern decomposition finds catalog garments that match that fit profile. 360° view on their mannequin.

Phase 3 — NDF

Neural Distance Field Inference

NDF trained on precomputed simulations enables sub-second fit prediction for any user body. Replaces interpolation with learned physics.

Phase 4 — Body Scan Moat

Styku Partnership + Van Fleet

Professional scanner integrations. Pop-ups at events. Mobile scanning vans in the Bay Area. Sub-5mm body accuracy unlocks "trust exactly what you see" fidelity.

Engine Decision

CPU → HOOD on Kaggle

Phase 1 — Keep

CPU XPBD

Pure Python numpy/scipy. Runs locally on M3 Mac. 284 tests validate it. Used for all local development and unit testing. Never removed.

Phase 2 — Build Now

HOOD / ContourCraft

MIT licensed. GNN-based cloth simulation. Accepts pre-stitched OBJ meshes. Runs on Kaggle free T4 GPU (30hrs/week). Handles seam-stitched garments natively. Visually realistic drape.

Validation Gate

XRTailor FEM

Apache 2.0. 15.1 Colab units reserved. Used as accuracy benchmark: run HOOD and XRTailor on same input, measure delta. If ≤2mm, HOOD is production-ready.

Simulate once offline. Serve instantly online. Interpolate fit for any user body from nearest reference results.

The 20 reference body grid must span the realistic distribution — not randomly sampled. Grid across chest circumference and shoulder width, with height variation.

Build Discipline

The one rule, always.

Map the full pipeline requirements before touching any implementation.

Before writing a single line of code, write down every input and output the component needs to support. Then read the API of whatever you're about to build and verify it can actually do those things.

Five minutes of verification before building saves months of rework.
