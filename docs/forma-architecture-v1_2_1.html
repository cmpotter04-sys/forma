01b // MVP Body Strategy

Synthetic Mannequin First

// Why
Sculptor Deferred to Phase 2
Real body scanning from consumer photos introduces the only genuine unknown in the architecture — 2-photo mesh fusion from uncalibrated cameras has no direct open-source comparable. Rather than blocking the physics pipeline on a research spike, we replace the Sculptor with a single mathematically perfect body mesh for Phase 1. The Geometer, Tailor, Archivist, and API are all completely indifferent to how the mesh was produced.
mesh formatbody_mesh.ply — identical to Sculptor output
scan_methodsynthetic_mannequin
accuracy_mm0 — perfect by definition
confidence1.0 — no uncertainty
generated viaSMPL-X shape params (pygarment)
// What You Gain
Three Strategic Advantages
1. Physics validation is clean. With a perfect mesh, any error in simulation output is the physics engine's fault — not scanning noise.

2. Retailer demo works immediately. Retailers think in size charts. Showing size M fits a size M mannequin and size S doesn't is exactly the right demo.

3. Sculptor drops in later with zero downstream changes. The output contract is already defined. Switching from synthetic to real scans is a body_source enum value change, not an architecture change.
Phase 2SAM 3D Body + height input + mesh fusion
Phase 3Forma Suit ArUco V2 (±1–2mm)
02 // Agent Roster

The Seven Agents

// Collision Engine
The Geometer
Drapes GarmentCode JSON pattern panels over the body mesh using XPBD physics simulation via NvidiaWarp-GarmentCode. Measures mechanical strain per body region at convergence. Hardware-agnostic. Confidence is 1.0 for synthetic mannequin; empirically derived f(scan_accuracy_mm, strain_magnitude, body_region) for photo-based bodies.
INbody_mesh.ply + seam_manifest.json
OUTfit_verdict.json + clearance_map + confidence
STACKPython, NvidiaWarp-GarmentCode, trimesh, scipy
MODELNo LLM — pure physics math
LICENSEApache 2.0 port in progress
🟢 Green

The Pattern Maker
Generates parametric sewing patterns using pygarment (GarmentCode, ETH Zurich). 115,000 patterns across all major garment categories from GarmentCodeData — no custom Master Blocks needed for MVP. Future upgrade: ChatGarment (CVPR 2025) for VLM-based pattern generation from product images.
INgarment_type, size_spec, fabric
OUTpattern.json + raw_panels.dxf

What You Cannot Delegate

These four things must stay with the founder
01Go/no-go decisions — does the XPBD clearance map look physically correct? Does the Phase 2 mesh fusion spike produce trustworthy results? Only you can judge this.
02Retailer conversations — no AI closes your first contract. Relationship, trust, reading the room.
03Spec quality — vague specs produce vague code. Specification writing is your primary technical skill in this model. Invest in it.
04Architecture changes — when something doesn't work, diagnosing whether it's a physics problem, schema problem, or integration problem requires architectural judgment. That's Layer 1.

Key Insight
The moment you have a working POST /verify endpoint returning a real fit verdict on a real garment — even with a synthetic mannequin — that is the moment to start retailer outreach. Not after Phase 2. Not after real body scanning works. That demo, with a mannequin, is enough to start the conversation and compress the sales cycle.

The Core Engine

Week 1: Validate clearance maps against tape-measure truth. Size S on size M → red. XL → green ease.
Week 2: Physics hardening — extended size range, second garment type, fabric sensitivity. Go/no-go gate.
Week 2: Wire programmatic pipeline function for batch processing.
Week 3: Archivist Supabase schema — body_source field from day one.
Week 4: REST API on Vercel — POST /verify → fit_verdict.json. Begin retailer outreach.

The Ledger + Real Bodies
Week 5: Sculptor V1 spike — SAM 3D Body + height input + 2-photo fusion. Go/no-go gate.

Expand Pattern Maker to 5 garment categories via GarmentCodeData.
Build Fabric Library — 8 material types with XPBD stiffness params.
Shopify integration POC — "Verified Fit" badge.
Complete Apache 2.0 Warp port before first revenue.
